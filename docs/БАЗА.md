# Часть I. Описание проекта

## 1. Коротко о продукте

БАЗА превращает обычное текстовое описание «кого вы ищете» в готовую базу B2B-лидов с контактами.

Вы пишете свободным текстом, например: *«Продаю кормовые добавки для КРС, нужны фермерские хозяйства Сибири от 200 голов»* — и сервис сам:
1. понимает нишу, регион, сегменты и коды ОКВЭД;
2. собирает подходящие компании из карт, реестров и веба;
3. находит email, телефон, адрес и проверяет их;
4. оценивает каждую компанию по релевантности (0–100);
5. отдаёт результат карточками/таблицей, с экспортом в Excel/CSV и выгрузкой в CRM.

Никаких фильтров, форм и ручного парсинга — свободный текст и одна кнопка «Собрать лиды».

## 2. Для кого и какую проблему решает

**Для кого:** отделы продаж и предприниматели в B2B, которым нужен поток холодных лидов по конкретной нише и географии (производство, опт, услуги, логистика, агро и т. д.).

**Проблема:** ручной поиск клиентов по 2ГИС/реестрам/сайтам — это часы рутины, разрозненные таблицы и «грязные» контакты. БАЗА автоматизирует весь цикл: поиск → контакты → проверка → приоритизация → выгрузка.

## 3. Как это работает — конвейер

```
Свободный текст
      │  (AI-разбор промпта)
      ▼
Стратегия: ниша · гео · сегменты · ОКВЭД
      │  («Собрать лиды»)
      ▼
Сбор из источников: 2ГИС · Яндекс Карты · ЕГРЮЛ · веб
      │  (+ дедупликация, + переиспользование склада)
      ▼
Обогащение: email / телефон / адрес  (+ проверка MX)
      │
      ▼
Скоринг релевантности 0–100
      │
      ▼
Работа с лидами: карточки/таблица, статусы, теги, заметки
      │
      ▼
Экспорт (CSV/Excel) · Вебхук в CRM (Bitrix24/AmoCRM)
```

Каждый запуск сбора параллельно пополняет общий **склад компаний** (РЕЕСТР БАЗА) и при следующих поисках переиспользует его — это удешевляет и ускоряет сбор.

## 4. Источники данных

| Источник | Что даёт | Примечание |
|---|---|---|
| **2ГИС** | компании, телефон, адрес | основной источник по РФ |
| **Яндекс Карты** | компании, карты | только на тарифах **Pro / Business** |
| **ЕГРЮЛ (Rusprofile)** | юрлица, ИНН, реквизиты | высокая достоверность, без контактов |
| **SearXNG (мета-поиск)** | сайты компаний из веба | агрегатор поисковиков |
| **Bing** | веб-поиск | резервный источник |
| **РЕЕСТР БАЗА (склад)** | ранее найденные компании | без затрат на API |

Поисковый запрос строится из связки **ниша + география + сегменты**. Коды ОКВЭД сейчас сохраняются и показываются в карточке проекта (прямые запросы в ФНС по ОКВЭД — в планах, Фаза 2).

## 5. Скоринг лидов (0–100)

Каждый лид получает оценку релевантности. Базовая логика:

- **База:** 35 баллов.
- **Сигналы контактности:** +домен, +email, +телефон, +адрес.
- **Совпадение по нише** (ключевое слово в названии/домене) и **по сегментам** (одно/несколько) — бонусы.
- **Штрафы:** нет контактов, агрегатор/каталог, демо/фолбэк.
- **Вес источника:** 2ГИС/Яндекс/ЕГРЮЛ/веб учитываются с разным весом.
- Итог ограничен диапазоном **0–100**.

Цветовая индикация в интерфейсе: **≥80** — мятный (топ), **60–79** — янтарный, **<60** — приглушённый.

Дополнительно работает фильтрация «покупатель, а не конкурент»: правиловая (отсев Wikipedia, форумов, объявлений «куплю/продам», магазинов-конкурентов) и, на платных тарифах, AI-фильтр.

## 6. Обогащение контактов

Обогащение добавляет к найденным компаниям контакты:

- **Email** — со страницы сайта, с проверкой MX-записей домена. Статусы: `valid` (MX есть), `no_mx` (MX нет), `syntax` (некорректный), `skipped` (DNS не ответил).
- **Телефон** — с сайта и из карточки 2ГИС.
- **Адрес** — сайт / 2ГИС / ЕГРЮЛ.

Если на сайте контактов не нашлось, БАЗА автоматически пробует карточку 2ГИС по названию+городу (закрывает значительную долю «пустых» лидов).

Обогащение запускается **автоматически** после сбора (если добавились лиды) и **вручную** — кнопкой «Обогатить» или для выбранных лидов в таблице.

> Извлечение ЛПР (конкретного лица, принимающего решение) — в планах; сейчас не реализовано.

## 7. РЕЕСТР БАЗА — склад компаний

Все компании, которые когда-либо находились (по всем проектам и организациям), складываются в общую таблицу `companies`:

- ключ дедупликации — базовый домен либо `название|город`;
- хранятся домен, контакты, город/регион, категории, ниши, источники, ИНН, 2ГИС-ID, лучший score, счётчик «сколько раз встречали», даты первой/последней встречи;
- при сборе данные **дописываются** (пустые поля заполняются, ниши/источники накапливаются, score берётся максимальный);
- при новом поиске склад отдаёт совпадения **бесплатно**, добирая квоту без обращения к платным API.

В карточке лида это видно в блоке **«РЕЕСТР БАЗА»**: найдена ли компания в реестре, сколько раз встречалась, в каких нишах/источниках, ИНН.

## 8. AI и его лимиты

AI используется в двух местах:
1. **Разбор промпта** — превращает свободный текст в нишу/гео/сегменты/ОКВЭД и поясняет стратегию.
2. **Фильтр кандидатов** — отсекает конкурентов/продавцов, оставляет покупателей.

Модели: YandexGPT → GigaChat → Anthropic (с фолбэком на правила, если AI недоступен). Чтобы один аккаунт не «съел» общий бюджет, на AI действует **месячный денежный лимит** по тарифу (см. ниже). На **Free** AI отключён — работает только правиловая логика.

## 9. Тарифы и лимиты

Цены — рублей в месяц. Лимиты сбрасываются 1-го числа.

| Параметр | Free | Starter (3 900 ₽) | Pro (12 900 ₽) | Business (34 900 ₽) |
|---|---|---|---|---|
| Проектов | 1 | 5 | 20 | 100 |
| Пользователей | 1 | 3 | 10 | 50 |
| Лидов в месяц | 0 | 5 000 | 25 000 | 100 000 |
| Сборов («Собрать лиды»)/мес | 0 | 30 | 100 | 300 |
| Лимит AI/мес | — | 300 ₽ | 3 000 ₽ | 15 000 ₽ |

Оплата — через **ЮKassa** (RU). Продление сейчас ручное (без автосписания); подписка активна до конца оплаченного периода.

## 10. Команды, роли, мультиаккаунт

Данные изолированы по организациям (мультитенантность). Внутри организации — роли:

| Роль | Возможности |
|---|---|
| **Владелец** | всё: тариф, лимиты, участники, вебхук, удаление организации |
| **Админ** | проекты, сбор, приглашения, вебхук, журнал |
| **Участник** | просмотр проектов и лидов |

Отдельно есть **глобальный администратор платформы** (`is_admin`) — доступ к админ-панели всего сервиса (не привязан к организации).

## 11. Безопасность и соответствие 152-ФЗ

- Хранение данных в РФ; оператор — ООО «ПРО ЛЕС».
- Пароли — bcrypt; доступ по JWT (короткий access + refresh-cookie), отзыв токенов при смене пароля/выходе.
- Подтверждение email (опционально, ст. 14).
- **Право на доступ:** выгрузка своих персональных данных (JSON) в «Настройки → Конфиденциальность».
- **Право на удаление:** удаление аккаунта с подтверждением паролем (каскадно удаляет организацию-владельца, проекты, лиды).
- **Срок хранения лидов:** по умолчанию 730 дней, далее автоочистка.
- Контроль вебхука ЮKassa по IP + перепроверка платежа на стороне сервера (защита от подделки).

## 12. Технологии и инфраструктура

- **Frontend:** Next.js 14 (App Router), тема «Cinematic Glass» (тёмная, мятный акцент).
- **Backend:** FastAPI + Celery (worker + beat) + SQLAlchemy 2.0.
- **Хранилища:** PostgreSQL (данные), Redis (очереди, токены, кэш, rate-limit).
- **Поиск:** SearXNG (self-hosted мета-поиск).
- **Платежи:** ЮKassa. **Уведомления:** email (SMTP), алерты в Slack/Telegram.
- **Инфраструктура:** Docker Compose за nginx, сервер Timeweb, домен usebaza.ru.

# Часть II. Руководство пользователя

## 0. Главное перед стартом

- Новый аккаунт создаётся на тарифе **Free**, где **сбор лидов недоступен**. Чтобы собирать базы — оформите **Starter/Pro/Business** (раздел 13).
- Весь путь: **Регистрация → Создать проект → (оплатить тариф) → Собрать лиды → Обогатить → Работать с лидами → Экспорт/CRM**.
- Демо-данные на дашборде (когда нет проектов) и на лендинге — это пример, помеченный «демо-данные».

## 1. Регистрация и вход

1. Откройте **usebaza.ru** → «Зарегистрироваться» (`/register`).
2. Заполните: **Имя**, **Название организации**, **Email**, **Пароль** (от 8 символов), согласие с условиями и политикой.
3. После регистрации:
   - если включено подтверждение email — придёт письмо (см. п. 2);
   - иначе — сразу попадёте в дашборд.
4. Вход — на `/login` (email + пароль). Забыли пароль — «Забыли пароль?» → письмо со ссылкой сброса.

## 2. Подтверждение email

Перейдите по ссылке из письма (`/verify-email?token=…`) — email подтвердится автоматически, и вас перенаправит на вход.

## 3. Дашборд

Главный экран после входа:

- **Карточка воркспейса:** название организации, тариф, ваша роль, шкала квоты лидов (`использовано / лимит`, цвет: зелёный <70%, янтарный 70–90%, красный ≥90%).
- **Плитки статистики:** проектов, лидов собрано, обогащено, активных сборов.
- **Предупреждение о квоте** при ≥80% — со ссылкой на тарифы.
- **Онбординг-превью** (если проектов нет): пример таблицы лидов и кнопка «Создать первый проект».
- **Список проектов:** карточки со статусом (новый/в работе/готово/ошибка), нишей, гео, счётчиками и источниками.

## 4. Создание проекта (промпт → стратегия)

1. Нажмите **«Новый проект»** → откроется окно **«Найти клиентов»**.
2. **Шаг 1 — Промпт.** Опишите свободным текстом, кого ищете (до 2000 символов; можно надиктовать голосом). Нажмите **«Анализировать»**.
3. **Шаг 2 — Стратегия поиска.** AI вернёт и предложит к редактированию:
   - пояснение и список целевых типов клиентов;
   - **Название проекта**, **Ниша**, **Регион/город**, **Целевые сегменты**, коды **ОКВЭД**.
   Проверьте, при необходимости поправьте.
4. Нажмите **«Создать проект»** — проект появится в списке.

> Совет: чем конкретнее промпт (продукт, кому продаёте, регион, размер, признаки готовности), тем точнее стратегия и выше качество лидов.

## 5. Сбор лидов

1. Откройте проект → панель действий вверху.
2. Нажмите **«Собрать лиды»** (по умолчанию до 500 за запуск). Задача встанет в очередь, статус виден во вкладке **«История задач»** и в реальном времени.
3. По завершении лиды появятся во вкладке **«Лиды»**, владельцу уйдёт email-уведомление.

> На Free кнопка сбора упрётся в квоту (0 лидов) — нужен платный тариф.

## 6. Обогащение

- **Автоматически:** запускается после сбора, если добавились лиды.
- **Вручную:** кнопка **«Обогатить»** (до 200 за раз) — добирает контакты для необогащённых.
- **Выборочно:** в режиме **Таблица** отметьте нужные строки и запустите обогащение по выбранным.

После обогащения у лидов появляются email (со значком статуса MX), телефон, адрес; score пересчитывается.

## 7. Работа с лидами

Во вкладке **«Лиды»**:

- **Поиск** — по компании, сайту, городу.
- **Фильтры** (кнопка «Фильтры»): статус, сортировка (по score/дате/компании, по убыв./возр.), диапазон score, наличие email/телефона. Кнопка «Сбросить фильтры».
- Переключатель вида **Карточки | Таблица**.

**Карточка лида** показывает: название + score, домен · город, шкалу score, бейджи статуса и источника, email и телефон (с пометками «найден/валиден»).

**Таблица:** Компания · Город · Телефон (✓/—) · Email (✓/—) · Score; есть массовое обогащение выбранных.

**Карточка компании (правая панель)** — по клику на лид:
- кольцо score, описание;
- контакты (email с MX-статусом, телефон) с кнопкой копирования;
- категории;
- **РЕЕСТР БАЗА**: найдена ли в складе, сколько раз, ниши/источники, ИНН, 2ГИС-ID;
- **теги**, **заметки**, **статус**.
Закрытие — Esc или клик по затемнению.

## 8. Статусы, теги, заметки

- **Статус** лида: `Новый → Контакт → Квалифицирован` (или `Отклонён`) — меняется в карточке компании.
- **Теги** — произвольные метки (добавить/удалить).
- **Заметки** — свободный текст по лиду (сохраняется автоматически).

## 9. Экспорт в CSV / Excel

В панели действий проекта — кнопки **Excel** и **CSV**. Выгрузка содержит: компанию, город, сайт, домен, email, телефон, адрес, score, статус, контакты. Большие проекты выгружаются потоково (без зависаний).

## 10. Интеграция с CRM (вебхук)

1. «Настройки → Организация» (роль Владелец/Админ) → поле **URL вебхука**.
2. Укажите endpoint вашей CRM (**Bitrix24**, **AmoCRM** или собственный).
3. При обогащении каждый лид отправляется в CRM (компания, город, email, телефон, адрес, сайт, score, статус, теги, проект). Есть повторные попытки при сбое.

## 11. Автосбор по расписанию

В проекте — блок **«Автосбор»** (роль Владелец/Админ):
- переключатель **Включён/Выключен**;
- расписание из пресетов: каждый день 9:00, каждый понедельник 9:00, 1-е и 15-е число 9:00, первое число месяца.
При срабатывании сбор и обогащение идут сами, по готовности придёт email.

## 12. Команда: приглашения и роли

«Настройки → Участники / Приглашения» (Владелец/Админ):
- создайте приглашение по email → получите ссылку-приглашение;
- приглашённый регистрируется/входит по ссылке и попадает в вашу организацию;
- роли участников меняются в таблице (см. матрицу ролей в Части I, п. 10).

## 13. Тарифы и оплата

1. Откройте **«Тарифы»** (`/plans`).
2. Выберите план → **«Перейти на …»** → переход на оплату **ЮKassa**.
3. После оплаты вернётесь на страницу подтверждения (`/billing/return`), где статус подтянется автоматически (обычно 2–5 сек). После активации лимиты применяются сразу.

Сменить тариф позже можно из «Настройки → Организация → Смена тарифа».

## 14. Настройки и приватность

`/dashboard/settings`:
- **Профиль** — имя, email, смена пароля.
- **Организация** — тариф, квоты, URL вебхука CRM.
- **Участники / Приглашения** — управление командой.
- **Журнал** — лог действий (Владелец/Админ).
- **Конфиденциальность** — **скачать свои ПД** (JSON) и **удалить аккаунт** (с подтверждением паролем). Здесь же контакты DPO и оператора.

## 15. Админ-панель

Только для глобальных администраторов (`/dashboard/admin`):
- **Обзор** — пользователи/организации/проекты/лиды, **реальная выручка (MRR по активным подпискам)**, распределение по тарифам, последние регистрации и сборы.
- **Пользователи** — назначить/снять админа, удалить.
- **Организации** — сменить тариф, изменить лимиты проектов/лидов.
- **Задания** — все задачи сбора/обогащения.
- **Логи** — журнал действий платформы.

## 16. Частые вопросы и сценарии

- **Зарегистрировался, а «Собрать лиды» не работает.** Вы на Free (0 лидов). Оформите Starter/Pro/Business.
- **Лидов нашлось мало.** Уточните промпт/нишу/гео, добавьте сегменты, повторите сбор; часть добирается из склада.
- **У лида нет email/телефона.** Нажмите «Обогатить»; если контактов нет в открытых источниках — поле останется пустым (помечено «—»).
- **Что значит ✓ у email.** Домен прошёл проверку MX (почта технически принимается).
- **Нужны Яндекс Карты как источник.** Они доступны на Pro/Business.
- **Как отдать лиды в CRM.** Укажите URL вебхука в настройках организации — лиды уйдут при обогащении.
- **Типичный сценарий:** создать проект → оплатить Pro → «Собрать лиды» → дождаться обогащения → отфильтровать по score ≥ 70 и «с email» → выгрузить Excel или включить вебхук в CRM → включить Автосбор на еженедельный режим.

## Деплой на прод

Сервер Timeweb, репозиторий `/opt/baza` на ветке `main`:

```bash
ssh root@72.56.11.69
cd /opt/baza && git pull && ./deploy.sh
```

`deploy.sh` собирает контейнеры, применяет миграции (`alembic upgrade head`), мягко перезагружает nginx и делает smoke-тест. Секреты — в `/opt/baza/.env.production` на сервере (в репозиторий не коммитятся). Неудачная сборка безопасна — старые контейнеры продолжают работать.

> Примечание по диагностике: если SSH/домен таймаутят с вашей машины, но `curl` по IP работает — это флапающий сетевой путь до сервера, а не падение прод-сайта. Проверяйте здоровье **с самого сервера** (`uptime`, `docker ps`, `curl https://usebaza.ru` локально на сервере).
